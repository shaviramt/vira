GitHub Pages.

Apa itu GitHub Pages?

GitHub Pages adalah layanan hosting untuk website statis yang bisa ambil file (HTML, CSS, JavaScript) langsung dari repository di GitHub, lalu run file tersebut melalui proses pembuatan hingga menjadi website. Untuk mengetahui apakah website dibuat memakai GitHub Pages yaitu dengan melihat akhiran pada URLnya, yaitu terdapat github.io.

Apa Keuntungan Menggunakan GitHub?

1. Tidak perlu setup server karena GitHub sendiri yang akan menjadi host GitHub Page kita.

2. Tidak perlu pusing memikirkan keamanan karena GitHub Page hanya memperbolehkan kita untuk menggunakan beberapa tools saja.

3. Cepat deploy karena kita tinggal membuat repository dengan nama username.github.io lalu ketikkan kode HTML atau Markdown pada laptop kemudian push ke repository tersebut.

Gimana Cara Membuat GitHub Page?

Langkah-langkah membuat github page:
1. Download Git Bash, setelah itu run code dibawah ini

    git config --global user.name "nama kalian"

   git config --global user.email "nama_email@email.com"

    Code di atas berfungsi untuk set username dan email kalian

2. Siapkan kerangka website sesuai keinginan, bisa menggunakan template yang tersedia online ataupun berkreasi sendiri sesuka kalian.

3. Pastikan kalian telah memiliki akun GitHub. Kalian bisa membuat akunnya disini.

4. Membuat repository baru dengan nama repository:

    username.github.io

".
5. Membuka Git Bash lalu clone repository GitHub kalian dengan cara

    "git clone https://github.com/username/username.github.io"

    Klik Enter !

6. Masukkan file kerangka website kalian (HTML, CSS, JavaScript) ke folder yang baru saja kalian buat di atas.

7. Pindah ke repository local kalian dengan cara

    "cdusername.github.io"


8. Kita mau masukkan file-file ini ke staging area. Caranya begini:

    "git add --all"

     Nah dengan code tadi, kita sudah berhasil masukkin seluruh file ke staging area.

9. Commit file-file yang ada di staging area. Untuk commit bisa dilakukan dengan cara

    "git commit --m "(bisa kalian isi dengan pesan bebas)"

     Nah, code barusan itu berfungsi untuk membuat record perubahan apa saja yang sudah kita lakukan.

10. Setelah commit, kita harus memasukkan file-file tadi ke repository kita di GitHub, dengan cara

    "git push -u origin main"

     Code tadi akan meng-update repository GitHub kita dan menyamakannya dengan repository local kita.

11. Refresh repository kalian di GitHub! Kalo semua file udah keupload di sana, langsung deh coba     buka website baru kalian di browser!
